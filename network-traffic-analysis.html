Network Traffic Analysis Lab

A practical cybersecurity lab focused on analyzing packet captures with Wireshark to identify suspicious traffic patterns, detect anomalies, and document findings.

Project 01 Network Security Traffic Analysis Wireshark

Project Overview

This lab demonstrates my practical understanding of network traffic analysis using Wireshark. The goal was to inspect captured packets, identify unusual behavior, and recognize indicators of suspicious activity within a simulated environment.

Tools Used

  • Wireshark
  • TCPdump
  • Basic TCP/IP knowledge
  • Packet inspection techniques

Focus Area

Network Security and Threat Detection through packet analysis, anomaly observation, and basic investigation workflow.

What I Did

  • Opened and inspected packet capture files in Wireshark
  • Reviewed traffic flow and protocol behavior
  • Looked for unusual communication patterns and anomalies
  • Documented suspicious indicators found during the analysis

Outcome

I successfully identified simulated suspicious traffic and strengthened my ability to observe network behavior, interpret packet-level activity, and document findings clearly.

What I Learned

This lab improved my understanding of how network traffic analysis supports cybersecurity investigations. It also helped me become more confident using Wireshark to detect anomalies, trace communication behavior, and think more critically about suspicious activity in a network environment.
